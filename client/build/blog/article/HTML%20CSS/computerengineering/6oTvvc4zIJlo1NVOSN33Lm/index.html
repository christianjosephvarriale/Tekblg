Basic structure

Let's first learn about the basics behind HTML. Here's a simple skeleton of a page

<!DOCTYPE html>
<html lang="en">    
   <head>        
      <link href="path/to/stylesheet" rel="stylesheet">
      <title>Website Title</title>  
    </head>
    
    <body></body> 
   
    <script src="path/to/script"></script>
</html>

Awesome. So a couple things to notice. We have everything in a <tag></tag> format. Whenever you use an HTML tag, you must use a closing tag. A closing tag is in the form of </tag>. You can also have tags within tags, meaning a tag may have children. This is important, because it allows us to implement styling, particularly positioning so you can control where elements render on the site.

Here we have boiler plate. Line 1 is required for all HTML documents to let the server know what type of file this is. This is to distinguish from XML which is a similar structure but gives a programmer the ability to specify what elements on a page are. While you may know you built a signup sheet, the browser is dumb. It just takes the code and renders it. Sometimes you got to help a dumb browser out.

The <head></head> tag controls some properties of the page, and is the first thing to render. As a result, if we need a resource to be loaded early, for example external code to style the page properly upon initial render, we'd do it here. The <body></body> tags allow you to mainly define the visible content on the browser, which we'll get to in a minute. Finally the <link> and <script> tags allow you to import Javascript or CSS files from either your directory or externally, very handy.

Of course there are many other tags available in HTML. Tags have many varying properties as how the <link> tag has the href="property" for example. If you're going to become a hacker, it's important to become comfortable with reading documentation, which is the next step after reading well written tutorial posts, and watching well transcribed videos. Check out the HTML section of W3 schools to satisfy your deepest desires: https://www.w3schools.com/html/default.asp

CSS

CSS is what makes webpages look beautiful especially if they're well designed like tekblg.com. It's what makes a page go from this:

To this:

We can we we defined things like:

  1. Where elements were positioned on a page

  2. How elements looked including colors, bolding, effects

In order to do this, we define classes in CSS. These classes create rules for how elements are supposed to look. Let's say I want to build a button, and that button should look like this button, and have a cool hover effect when I hover over it:

For reference, here is how it is defined in the HTML

So we have a <button> tag with some properties, one being class="class1 class2 ..." and other being style="style: value". We can define our styling in the html directly, or in an external .css file. Here, we used styles like font-weight: 600 to specify how thick the text is, border: 2px solid rgb(244, 48, 68) to specify the outline should be red, with a 2 pixel think border. We then defined a child <span> tag to contain the button text. This is a matter of taste, do you like fine caviar or well aged wine? Both are great. We could have defined a structure like:

<button class="class1... " style="style1: value1... ">
   <div>Button Text</div> 
</button>

<div>Button Text</div>

In general the most common tag you will see is <div> think of it like a general tag. It could store or represent any type of element, but calling it <button> or <input> makes the HTML more readable and often also you to leverage properties unique to that tag.

If we wanted to implement external styling, we might define a class and :hover pseudoclass thanks for the fancy word like so:

.MuiButtonBase-root {
   border: 2px solid rgb(244, 48, 68);
   font-weight: 600;
   background-color: white;
}

.MuiButtonBase-root:hover {
   background-color: #fff;
}

Notice how we used a combination of rgb() and #hex to represent colors. Both are supported by major browsers. we can also see the . before the className, indicating it is a class. We can also do things like apply styling to unique elements using id or selecting elements based on conditionals like apply this style to all div's which are children of a p tag of class awesome class. Note, I used a /* here's a comment, it won't be read by the browser */ to add a comment for readability or to explain things. this would be something like:

p[class="awesome"] div {
   /* awesome styles... */
}

Of course, to learn all the available features be sure to checkout: https://www.w3schools.com/css/default.asp

Build process

STOP RAMBLING AND SHOW ME THE MONEY. First off, we have no money. Secondly, great idea. Now that you get what's going on, let's dive into building out that signup form

First, we're going to start off with the head tag. We'll define the document type, make some imports using the <link> tags for Google font's and Bootstrap Stylesheets to leverage predefined classes, and use the <meta> tags which is helpful for Google SEO and the browser optimization.

<!DOCTYPE html>    
   <html lang="en">
      <head> 
        <meta charset="utf-8">  
        <meta http-equiv="X-UA-Compatible" content="IE=edge"> 
        <meta name="viewport" content="width=device-width, height=device-height, user-scalable=0, initial-scale=1, minimum-scale=1.0, maximum-scale=1.0">      
        <meta name="apple-mobile-web-app-capable" content="yes">
        <link href="./stylesheet.css" rel="stylesheet">
        <link href='https://fonts.googleapis.com/css?family=Open+Sans:400,300,300italic,400italic,600' rel='stylesheet' type='text/css'> 
        <link href="https://netdna.bootstrapcdn.com/font-awesome/3.1.1/css/font-awesome.css" rel="stylesheet">      
        <title>Signup Form</title> 
   </head>

  <body></body>
</html>
