Some helpful tips on designing UI

  • 2020-01-03T00:00-05:00
  • By Tekblg

Let's start off with the basics

If you have never designed a great looking UI before that's fine. Other people have and you can learn from what already works.

Let's not reinvent the wheel with our designs, but copy what has already been proven effective in industry

If you're struggling to learn something the quickest way to ensure you're on the right track is to dedicate to noob status and use the internet. Google whatever you're designing whether it'd be a dashboard or responsive component and see what's available. Code is great because it's so accessible and therefore easy to improve upon into your own design ( which by the way might constitute possible legal infractions if the code has been found under certain licenses, so be sure to check ). With UI however, this is generally a smaller risk as opposed to replicating a highly novel data processing algorithm for example.

The reality is that few people are actually naturally creative, and therefore many of these designs are probably originating from previous designs and so forth.

Key concepts

Margin and Padding are your friends

From a UI development perspective they are the same thing, whitespace. Whitespace makes a webpage look less cluttered and therefore makes information more digestible. If you have a lot of content, consider separating it into cards or different pages altogether. Having a long scrolling page is fine as long as the most captivating / relevant information is at the top. Whitespace also needs to be consistent throughout the page to make everything look even. Finally, consider centering your page for long blurbs of text, but using variance in the amount of space from the edges of the screen to the content

Mobile First design philosophy

In software industry everyone says mobile first but in reality we're developing using desktops and therefore we tend to think of that design first. That makes sense to us, because we have more real estate on the screen and our lazy minds don't like the constraints of a 350px width screen. Unfortunately in many cases such as blogs or web apps for younger people the mobile formatting will receive the most traffic and therefore should be optimized. Remember, things needs to take up the full size of the screen, expect long scrolling pages and using bigger calls to actions like buttons.

In general, the amount of content you can show is less and elements which were previously side to side should stack on top of each other. All our clients have loved the usage of cards on the mobile view. Make your designs simple and clean. Show little content and make sure the text is nice and readable. It's also difficult to format images especially when the width is much larger than the height and therefore you may have to crop the image to focus on the highlight ( maybe a happy customer in the image or a captivating section ). Who's doing it best

Let's analyze some of the top dogs:

Material UI

Material UI provides an impressive, well documented code base which is compatible with ReactJS and as a result powers many of the components used on this website ranging from the navigation bar to cards ( which we believe is the most visually appealing component ). If we look at the above example blog page, we can see the usage of a couple key components, cards, buttons, and parallax web design.

Let's start with the cards. Material UI integrates a header image with a white overlay and some text. The next is normally Roboto font although they have many other options available. Part of what makes the cards so appealing is the combination of a well optimized image which maintains proper aspect ratio, clever usage of icons ranging from settings to social media icons, and the roboto font which is characteristic of Google which we are accustomed to.

EOC

Buttons with material UI come in different varieties including with a border, without a border, and filled with a color. When you have text without some sort of border it isn't apparent to a user that it is clickable and therefore should be avoided. Buttons with a fill are a more classic approach as can be seen above, and material UI has a "ripple" Javascript feature which adds to the ::active functionality ( very cool ). Lastly, buttons with an outline are our personal favourite as one can invert the colors on ::hover which makes it very apparent the button is clickable. In addition, using outlines and in general thin borders on a design makes it look more elegant but less readable so it's important to keep them at around 2px as a rule of thumb.

Bootstrap

Bootstrap is the world's most popular design framework by far. It's easy to implement, with few features such as dynamic javascript but is sufficient for most designs. Part of the reason for its immense popularity is how clean the design is. We see the usage of a traditionally centered document with a fairly conservative use of margin for the above example pricing page. The footer is a series of unordered lists with texts which is industry standard for a company not looking to design a fancy, dynamic site and is indicative of a company that isn't necessarily in the tech sector.

One thing to note is the usage of a very traditional design with a header, description, call to action format which is all text-align: centered. It's interesting how this particular page uses a static navigation bar ( contrasted to a dynamic shadowing / resizing design which is more modern ), in combination to a left-aligned style of displaying lists. Generally in design you don't want to mix the two, which can be summarized by keeping your design consistent.

Foundation

Foundation is acclaimed to be the most advanced framework that is available. I think it's true they have unique components available that aren't present with google, semantic UI, or bootstrap such as vertical timelines or star rating blocks which are a result of contributions from multiple authors. Obviously, this risks the possibilities of defective components but we'll give them a plus 1 for that for now.

If we look above we see a template ecommerce site as taken from the foundation webpage. I don't know about you but we don't think this is elegant design. Right of the bat we see the aspect ratio of components is wonky, the site feels cluttered and elegance is nowhere to be found. It's a nice way to display information in a traditional sense, and to their credit an ecommerce site is a likely culprit to be overloading a user with information ( they are trying to sell you stuff and fill your mind with things you don't need after all ), but their are ways to improve this.

First off all there needs to be some sense of division between sections. If we have a carousel right under a list immediately following a bunch of descriptions you get very frustrated very quickly. Remember whitespace is your friend here, and when we mean whitespace we do mean it should be white or grey. Over-usage of color can make your site seem unprofessional ( especially a landing-page ). Look to a site like https://tophat.com/ for a great example of what effective white space looks like.
